初识React-Native

撰写于 2016-12-05 修改于 2016-12-05 分类 React-Native

React-Native是由Facebook开源的一套使用JS(JavaScript)构建NativeAPP的框架

初识背景

我从2015年9月开始学习iOS的APP开发，亲身感受的是原生开发的应用的完美，在我看来，只有你想不到的，没有开发人员做不到的。
然而，在这样的情况下，原生应用有其不可避免的劣势，最让用户以及开发人员头疼的就是更新迭代问题。
常规的APP更新的方式都是采用下载更新版本，而部分APP在用户使用过程中，会发现许多内容的更新并没有通过重新下载。而是类似于网页的在线更新模式，这就要归功于混合开发的模式了。
之所以下定决心遁入RN的坑是因为刚好进公司之后也一直在做webAPP，对于使用JS构建APP应用也有了自己的心得体会。当然也是给自己找点事做做，脑子太闲了就会笨的。

初识过程

自学某一门语言或者技术对于我来说已经不是那么难以下手了，在这个互联网高速发展的时代，想要些资料基本就和呼吸一样简单。
不得不说，RN的官网做的还是非常不错的，入门级使用以及一系列进阶使用技巧在一段时间里完全可以成为指南与工具书。

那么React-Native到底可以做什么：

  1. 可以基于 React Native使用 JavaScript 编写应用逻辑，UI 则可以保持全是原生的。这样的话就没有必要就 HTML5 的 UI 做出常见的妥协
  2. React 引入了一种与众不同的、略显激进但具备高可用性的方案来构建用户界面。长话短说，应用的 UI 简单通过一个基于应用目前状态的函数来表达。

在接触混合开发之前，我并不觉得这能让我感到兴奋，而当我完成了两个公司的项目之后，我渐渐地对RN产生了兴趣，或者说对这样一种code once, run anywhere的方式产生了兴趣。

后话

现在，让我来慢慢升入学习React-Native。

如果对于HTML5等还不是很熟悉，或者说没有接触过，不妨先去W3School、菜鸟教程等学习站点进行初步了解。
